User Accounts

A user is an entity, or figuratively an identity, in FOCUSA. This user can perform tasks like creating and viewing posts, creating and maintaining a profile, to even the trivial task of forwarding and sharing posts and attachments directly to their email.

This idea of managing users based on accounts, is something that improves the simplicity and flexibility of managing FOCUSA. The administrator has the ability of creating, updating, and deleting users directly through the dashboard.

User permission hierarchy

Not every user is allowed to perform every task. The access control of FOCUSA allows us to control the actions to the database, based on the user permssion tag. Every action performed to the database is well monitored, so if any attack ever occurs, the administrator will find out who did it by searching the logs. The user permission tag hierarchy is as follows:

Permission symbolTasks permitted
"g"

  • A Guest can view any post, uploading text-only posts(without any attachments), and follow people.
  • It is the user with the least priviledges.

"nr"

  • A News Reporter has all priviledges as the guest, along with the ability of uploading posts with attachments.
  • This user has moderate priviledges.

"a"

  • An Administrator has the most priviledges, along with the ability of modifying, deleting, and creating user accounts.
  • This user can also delete posts by any other user. They cannot edit any user's posts though.
